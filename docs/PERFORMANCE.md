# Performance Notes

阶段 5 已完成实现侧性能优化，但真实指标需要在目标机器和真实图片集上采样。

## Current Optimizations

- 合集详情页一次最多加载 10,000 张图片，仍使用虚拟滚动渲染可视行。
- Rust 图片分页上限、重复检测和 JSON 导出上限提升到 20,000 张。
- 缩略图行高会随设置中的缩略图尺寸变化，避免万级列表滚动错位。
- 查看器请求最大边 4096 的预览缓存，降低超大图直接解码造成的内存压力。
- 已缓存缩略图和查看器预览继续走磁盘缓存命中路径。

## Local Benchmark Fixture

不要提交测试图片。仓库已忽略 `fixtures/`。

```bash
pnpm bench:fixtures 1000
```

生成后在桌面应用中导入 `fixtures/bench-1000`，记录导入耗时、首次滚动、再次打开缓存图的耗时。

## Acceptance Targets

- 1000 张图片导入小于 30 秒。
- 已缓存图片打开小于 200ms。
- 万级图片列表滚动无明显卡顿。
- 超大图浏览不因一次性解码原图导致崩溃。

当前本机只完成构建和自动化测试验证；上述时间指标需要真实图片集和目标平台复测。
